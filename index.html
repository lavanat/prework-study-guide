HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • There are three different ways to style a webpage - inline CSS, internal CSS style sheet, and external CSS style sheet
  • External CSS style sheet is the most preferable.
  • The CSS rule is set up so that you first have the selector for what the style applies to, like h1, ul, etc. then list out the property and values in pairs in the declaration block that follows.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout branch-name: changes current branch to the branch-name
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A or git add .: add the changed files in our working directory to the staging area of Git.
  • git commit -m "message": commit changes to the repository with the message
  • git pull origin main: make sure the current branch is up to date before opening a pull request
  • git push origin branch-name: push up the changes to the working branch
  • Then, in GitHub, click on the "Pull Requests tab" and click on "New pull request" button. Make sure the base branch is main and the comparison branch is your working branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, code is executed from top to bottom.
  • However, using conditional statements such as if statements allows us to interrupt control flow.
  • The code to be executed should be written inside the curly brackets.
